前端三板斧(一)

目的

  • 知道前端到底是干啥的
  • 知道HTML+CSS是啥
  • 了解HTML+CSS的作用
  • 知道咋学HTML、CSS

前端到底是干啥的

  • 前端，在以前的概念里（尤其是前后端不分离的刀耕火种时代），做的其实就只是网页的实现（只包括pc端），后来随着各种诸如RN（React Native）、Flutter这样的移动端框架的出现，在各种终端的面向用户的界面设计也都被称作前端，“大前端”。

所以前端是所有程序员里最贴近用户的工种，所以也要求前端要有基本的产品思维和一定的审美，当然这些并不是决定性因素，有则当然好没有以后慢慢培养:P。

前端三板斧

  • HTML

即超文本标记语言（HyperText Markup Language），通俗点理解其实就是前端页面的容器，你可以把一个页面想象成一片沙滩，你想要往里放东西的话总要挖个坑出来，而HTML大多数时候就是你挖坑的那个铲子，在初学阶段你需要做的是把最基础的标签记下来、明白不同书写方式时他们的层级关系（嵌套、同级之类的），你就走过了HTML的第一个关卡。

  • CSS

层叠样式表（Cascading Style Sheets）
当你用HTML把坑挖出来了以后，就会发现它其实只是一个坑，里面的东西还需要我们去填充，并且我们还要求这些东西要足够好看，用户的交互感强。这时候就是CSS发挥作用的时机了。

在我现在看来CSS是前端学习中最难的一部分（要把它整得明明白白真的需要不小的努力），但好在它的学习曲线并不陡峭，你想要深入学习也许需要很多努力和一些灵光一闪，但它的入门并不会太难。

推荐的链接：

慕课网

w3school

菜鸟教程

这几个教程都有在线IDE（集成开发环境），选择其一扎扎实实地写下自己的代码，所见即所得。这个过程不建议看视频和看书，第一是浪费时间，第二是我认为如果没有亲手写点BUG出来并且亲自把它修好的话，你是体会不到编程的乐趣的。当然你如果有自己的学习方法，推荐你看《Head First HTML+CSS》这本书，字大图多好理解。视频的话，市面上的视频都是大差不差的，就不推荐了。

那么这次的博客先写到这，有空再写剩下的啦:O

开始写博客啦

前言

从前段时间开始我就想找个时间来经营起自己的博客，但是一直忙于校招没啥时间，前几天终于算是结束了秋招最后一场面试，这两天研究了一下，就开始了我的第一篇博客啦！

我的博客会写些什么

由于我早有想法写自己的博客，所以我做了一个短期的规划：

  • 1.写我在接触前端开发接近一年里，我是怎么学习前端开发的，我大概会写我从我还什么都不懂到独立完成一个项目的经历中所遇到的迷茫困难和学习路径，希望能为后来者提供一些参考，尤其是张老师项目组的学弟学妹们，我还是很希望你们可以在一年之后找到好的工作，学长也愿意为你们尽一些绵薄之力:P。
  • 2.我现在手头上的项目还有待改进，并且有可能是要让学弟学妹们接手的，所以我会尽量就我的理解写一些利于你们上手的文章，以求你们在看我的代码时，能少一些对学长技术水平不行的怨念= = 。
  • 3.还有一点就是我在学习的过程中会把自己的体会、心得、总结等写成文章发出来，也算是给自己的一个笔记本吧，可以预料到这部分的文章可能会出不少错误或者是个人片面的理解，但是我觉得年轻人还是要不怕犯错，写错了总比不写好，大不了重写一篇=_=。
